\section{NAME}\label{name}

\textbf{fun} --- transpile and run a \emph{fun} source file (a tiny
Lua-targeted source language with Python-style
\passthrough{\lstinline!:!} blocks and indent-closed bodies).

\section{SYNOPSIS}\label{synopsis}

\textbf{fun} {[}\emph{OPTS}{]} \emph{FILE} {[}\emph{ARGS}\ldots{]}

\section{DESCRIPTION}\label{description}

\emph{fun} is a Lua-targeted source language whose transpiler is a
single short Lua module (\passthrough{\lstinline!fun.lua!}). Source
files use the \passthrough{\lstinline!.fun!} extension.

\textbf{The transpiler is strictly line-by-line.} Each physical line is
parsed and rewritten in isolation. There is \textbf{no line-continuation
character} --- a backslash at end-of-line has no special meaning. The
only cross-line state is an indent stack used to emit closing
\passthrough{\lstinline!end!}s when a block's body outdents.

This has direct consequences:

\begin{itemize}
\tightlist
\item
  Conditions in \passthrough{\lstinline!if (...)!},
  \passthrough{\lstinline!elseif (...)!},
  \passthrough{\lstinline!while (...)!} \textbf{must close on the same
  line}.
\item
  Comprehensions \passthrough{\lstinline![expr for v in iter (if c)]!}
  \textbf{must fit on one line}.
\item
  Long expressions that don't fit on one line must be broken via
  intermediate locals, not by splitting one statement across lines.
\end{itemize}

By the time the source reaches Lua, it \emph{is} Lua. \emph{fun} adds a
small set of per-line rewrites that compile away --- no runtime layer.

\section{QUICK START}\label{quick-start}

\begin{lstlisting}[language=bash]
./fun foo.fun arg1 arg2
./fun -s foo.fun           # print transpiled Lua (no run)
./fun -h                   # help
\end{lstlisting}

\begin{lstlisting}[language=Fun]
#!/usr/bin/env fun
-- vim: ft=fun

greet := fun(name): !"hello, "..name
sq    := fun(x):    !x * x

print(greet("world"))
print(sq(5))
\end{lstlisting}

\subsection{\texorpdfstring{Block introducer
\texttt{:}}{Block introducer :}}\label{block-introducer}

\passthrough{\lstinline!:!} ends a header line and opens a block:

\begin{lstlisting}[language=Fun]
if (x > 0):
  print("pos")
elseif (x < 0):
  print("neg")
else:
  print("zero")
\end{lstlisting}

{\def\LTcaptype{none} % do not increment counter
\begin{longtable}[]{@{}ll@{}}
\toprule\noalign{}
Header & Lua \\
\midrule\noalign{}
\endhead
\bottomrule\noalign{}
\endlastfoot
\passthrough{\lstinline!if (c):!} &
\passthrough{\lstinline!if (c) then!} \\
\passthrough{\lstinline!elseif (c):!} &
\passthrough{\lstinline!elseif (c) then!} \\
\passthrough{\lstinline!else:!} & \passthrough{\lstinline!else!} \\
\passthrough{\lstinline!for X in Y:!} &
\passthrough{\lstinline!for X in Y do!} \\
\passthrough{\lstinline!while c:!} &
\passthrough{\lstinline!while c do!} \\
\passthrough{\lstinline!fun(args):!} &
\passthrough{\lstinline!function(args)!} \\
\end{longtable}
}

\passthrough{\lstinline!if!} / \passthrough{\lstinline!elseif!}
\textbf{require parens} around the condition.

\subsection{Indent-closed blocks}\label{indent-closed-blocks}

A line ending with \passthrough{\lstinline!:!} opens a block. The body
is the consecutive following lines at greater indent. The transpiler
emits \passthrough{\lstinline!end!} when indent returns to the header's
level (or below).

\begin{lstlisting}[language=Fun]
foo := fun(x):
  if (x > 0):
    print("pos")
    return 1
  return 0
-- foo's `end` emitted here automatically.
\end{lstlisting}

\subsection{One-liners (auto-end)}\label{one-liners-auto-end}

If content follows \passthrough{\lstinline!:!} on the same line, it's a
one-liner --- the transpiler auto-appends \passthrough{\lstinline!end!}:

\begin{lstlisting}[language=Fun]
sq    := fun(n): !n * n
greet := fun(s): !"hi "..s
if (x > 0): print("pos")
for i in xs: print(i)
\end{lstlisting}

becomes

\begin{lstlisting}[language=Lua]
local sq    = function(n) return n * n end
local greet = function(s) return "hi "..s end
if (x > 0) then print("pos") end
for _,i in ipairs(xs) do print(i) end
\end{lstlisting}

Auto-end is \textbf{skipped} if the line already contains
\passthrough{\lstinline!end!}. So:

\begin{lstlisting}[language=Fun]
sort(xs, fun(a,b): !a < b end)
\end{lstlisting}

The user-supplied \passthrough{\lstinline!end!} closes the inline
lambda; the call closes with \passthrough{\lstinline!)!}. No auto-end
injected.

\subsection{Inline lambdas in
expressions}\label{inline-lambdas-in-expressions}

Anonymous \passthrough{\lstinline!fun(args):!} inside a call/argument
list keep an explicit \passthrough{\lstinline!end!} (the call's
\passthrough{\lstinline!,!}/\passthrough{\lstinline!)!} continues the
expression; indent can't close it reliably):

\begin{lstlisting}[language=Fun]
sort(xs, fun(a,b): !a < b end)
table.sort(t, fun(a,b): !a[1] < b[1] end)
\end{lstlisting}

\subsubsection{\texorpdfstring{Tip: avoid the explicit \texttt{end} by
extracting the
lambda}{Tip: avoid the explicit end by extracting the lambda}}\label{tip-avoid-the-explicit-end-by-extracting-the-lambda}

Bind the lambda to a local first --- both functions then close via
indent, no explicit \passthrough{\lstinline!end!} anywhere:

\begin{lstlisting}[language=Fun]
-- ✗ inline lambda needs explicit `end` to close before `)`:
show := fun(tree):
  nodes(tree, fun(node, lvl, pre):
    p := ...
    io.write(...)
  end)

-- ✓ extract to a local — outdent closes both funs cleanly:
show := fun(tree):
  fn := fun(node, lvl, pre):
    p := ...
    io.write(...)
  nodes(tree, fn)
\end{lstlisting}

Same Lua output, less syntactic noise. Useful for any multi-line
anonymous fun passed as an argument.

\subsection{Return}\label{return}

\passthrough{\lstinline"!"} at line start (or after a
\passthrough{\lstinline!:!}) becomes \passthrough{\lstinline!return!}:

\begin{lstlisting}[language=Fun]
foo := fun(x): !x * 2          -- return x*2
bar := fun(x):
  if (x > 0): !x
  !-x
\end{lstlisting}

\subsection{Comprehensions}\label{comprehensions}

\begin{lstlisting}[language=Fun]
squares := [x*x for x in xs]
evens   := [x for x in xs if x % 2 == 0]
labels  := [t.name for t in tasks if t.done]
\end{lstlisting}

Compile to an inline IIFE. The transpiler uses balanced-bracket parsing,
so \passthrough{\lstinline!r[col.at]!}-style indexing inside expr works:

\begin{lstlisting}[language=Fun]
xs := [r[col.at] for _,r in ipairs(rows) if r[col.at] ~= "?"]
\end{lstlisting}

Iterator handling: - single loop var →
\passthrough{\lstinline!for \_,v in ipairs(iter)!} - comma in loop var →
\passthrough{\lstinline!for k,v in pairs(iter)!} - iter contains
\passthrough{\lstinline!(!} → passed through verbatim

\textbf{Comprehensions must fit on one line.}

\section{DIAGNOSTICS}\label{diagnostics}

When transpiled Lua fails to load, \passthrough{\lstinline!fun!} writes
a 5-line window around the offending \textbf{transpiled} line to stderr,
then errors. Line numbers track source closely (per-line transpile
preserves line count; comprehension expansion pads newlines).

To see the transpiled output:

\begin{lstlisting}[language=bash]
./fun -s file.fun
\end{lstlisting}

\section{RESTRICTIONS}\label{restrictions}

\subsection{Line-by-line --- no
continuation}\label{line-by-line-no-continuation}

The transpiler processes one source line at a time. \textbf{No
\passthrough{\lstinline!\\!} continuation, no implicit continuation.}
Each
\passthrough{\lstinline!if!}/\passthrough{\lstinline!elseif!}/\passthrough{\lstinline!while!}
header and each comprehension must fit on one line.

If a line gets too long, factor with intermediate locals:

\begin{lstlisting}[language=Fun]
-- ✗ too long, no way to wrap
result := some_long_call(arg_one, arg_two, ... arg_seven)

-- ✓ split via locals
args1 := (arg_one)
args2 := (arg_two)
result := some_long_call(args1, args2, ...)
\end{lstlisting}

Or for a long condition:

\begin{lstlisting}[language=Fun]
-- ✗
if (x > 0 and y > 0 and z > 0 and w > 0):
  body

-- ✓
all_pos := x > 0 and y > 0 and z > 0 and w > 0
if (all_pos):
  body
\end{lstlisting}

\subsection{\texorpdfstring{Comprehension iterator with \texttt{{]}}
inside}{Comprehension iterator with {]} inside}}\label{comprehension-iterator-with-inside}

Iterator content can't contain literal \passthrough{\lstinline!]!}
(e.g.~\passthrough{\lstinline!:gmatch"[\^,]+"!}). Bind the iterator
first:

\begin{lstlisting}[language=Fun]
-- ✗ broken
t := [thing(x) for x in s:gmatch"[^,]+"]

-- ✓
it := s:gmatch"[^,]+"
t  := [thing(x) for x in it]
\end{lstlisting}

\subsection{Indent rules}\label{indent-rules}

\begin{itemize}
\tightlist
\item
  Tabs count as 2 spaces (or pick consistent indent --- don't mix)
\item
  \passthrough{\lstinline!elseif!} / \passthrough{\lstinline!else!}
  continue the open \passthrough{\lstinline!if!} block (no premature
  close)
\item
  Block bodies must indent \textbf{more} than their header
\item
  Outdent below header indent closes the block
\end{itemize}

\subsection{\texorpdfstring{\texttt{!} substitutes
globally}{! substitutes globally}}\label{substitutes-globally}

\passthrough{\lstinline"!"} outside strings/comments becomes
\passthrough{\lstinline!return!}. So: - \passthrough{\lstinline"!="}
becomes \passthrough{\lstinline!return =!} --- broken. Use
\passthrough{\lstinline!\~=!}. - \passthrough{\lstinline"not !x"}
becomes \passthrough{\lstinline!not return x!} --- broken. Use
\passthrough{\lstinline!not x!}.

\subsection{\texorpdfstring{\texttt{:=} only at statement
start}{:= only at statement start}}\label{only-at-statement-start}

The \passthrough{\lstinline!:=!} pattern matches at the start of a line
(after optional whitespace). Don't use it mid-expression.

\subsection{\texorpdfstring{Method \texttt{:} vs block
\texttt{:}}{Method : vs block :}}\label{method-vs-block}

The \passthrough{\lstinline!:!} is a method-call when followed
immediately by an identifier (\passthrough{\lstinline!obj:method()!}).
It's a block introducer when followed by whitespace or end-of-line
(\passthrough{\lstinline!if (c):!},
\passthrough{\lstinline!fun(x): body!}).

\subsection{Plain tables have no
methods}\label{plain-tables-have-no-methods}

\passthrough{\lstinline!xs:remove(n)!} looks for a
\passthrough{\lstinline!remove!} field; standard Lua. Use
\passthrough{\lstinline!table.remove(xs, n)!}.

\subsection{\texorpdfstring{String hiding does not handle
\texttt{\textbackslash{}"}-escaped
quotes}{String hiding does not handle \textbackslash"-escaped quotes}}\label{string-hiding-does-not-handle--escaped-quotes}

The hiding regex is \passthrough{\lstinline!'"[\^"]*"'!}. Escaped quotes
break it. Use single quotes, \passthrough{\lstinline![[...]]!}, or split
via \passthrough{\lstinline!..!}.

\section{EXAMPLE}\label{example}

\begin{lstlisting}[language=Fun]
#!/usr/bin/env fun
-- vim: ft=fun

NUM := "Num"   -- type marker
Num := fun(txt, at):
  !{is=NUM, txt=txt or "", at=at or 0,
    n=0, mu=0, m2=0}

add := fun(num, v):
  num.n = num.n + 1
  d := v - num.mu
  num.mu = num.mu + d / num.n
  num.m2 = num.m2 + d * (v - num.mu)
  !num

mid := fun(num): !num.mu

adds := fun(values):
  out := Num()
  for _,v in ipairs(values): add(out, v)
  !out

print(mid(adds({1,2,3,4,5})))
\end{lstlisting}
